# Лаборатория №1
   Для выполнения лабораторной работы был определен порождающий паттерн проектирования **Абстрактная фабрика** (*Abstract Factory*) .

# Предметная область
   Создание персонажа в компьютерной игре, для которго мы настраиваем снаряжение.
   Персонаж может быть питомцем, NPC, боссом.

# Описание проблемы
    При создании персонажа есть следующие сложности:
    -Создание персонажа с дополнительными характеристиками, которые нужно настраивать поэтапно.
    -Создания различных вариантов персонажей, характеристики которых могут быть одинаковы, но снаряжение разное. (Например: босс с топором или щитом)

# Решение
   Использование паттерна **Абстрактная фабрика**. Абстрактная фабрика позволяет создавать фабрики, которые производят семейства связанных объектов без явного определения их классов.
    В данном случае:
-**Admin** управляет процессом создания персонажа, вызывая методы строителя по порядку.
-**Builder** определяет интерфейс для добавления снаряжения персонажу и дополнительных характеристик.
-**ConcreteBuilder** реализует конкретные способы сборки для разных вариаций персонажей (питомцем, NPC, боссом).
-**Product** — персонаж с определённым снаряжением.

-**AbstractFactory** — интерфейс фабрики, создающий семейство связанных объектов (персонажей).
-**ConcreteFactories** — конкретные фабрики для питомца, NPC, босса.
-**Product** — базовая абстрактная персонажская сущность.
-**Конкретные продукты** — реализуют конкретные типы персонажей.
-**Класс управления** — пример использования фабрик для создания персонажей.
